# music-oyakodon

Suno AIを使用して作成された、親子丼に関するAI生成楽曲のプレイリストを再生する、シンプルなセルフホスト型ウェブミュージックプレイヤーです。

## デモ

ライブデモはGitHub Pagesでご覧いただけます。

プレイヤーのインターフェースは、現在再生中のトラックのアートワークをぼかした背景の上に、プレイリスト、アルバムアート、楽曲情報、歌詞を3パネルレイアウトで表示します。

## 機能
- **3パネルUI**: プレイリスト、メインプレイヤーコントロール、歌詞を同時に表示します。
- **動的な背景**: 現在再生中の楽曲のアルバムアートを、ぼかしたフルスクリーン背景として使用します。
- **メディアセッションAPI**: ネイティブOSのメディアコントロールと連携し、ロック画面、通知センター、または接続されたデバイスからの再生管理を可能にします。
- **セルフホスト**: サーバーサイドの依存関係を持たず、静的なウェブページとして動作します。すべてのアセットはローカルに配置されます。
- **カスタマイズ可能なプレイリスト**: Suno AIからエクスポートされた任意のプレイリストに簡単に適応可能です。
- **複数のソート順**: `riffleShuffle` や `likeSort` など、プレイリストをソートまたはシャッフルするためのJavaScript関数をいくつか備えています。

## 使い方
ご自身のSuno AIプレイリストでこのプレイヤーを使用する手順は以下の通りです:

1.  **アセットのダウンロード**: SunoからプレイリストのJSONファイルと、関連するすべてのオーディオ（`.mp3`）および大きな画像（`.jpeg`）ファイルをダウンロードします。
2.  **ファイルの整理**:
    - すべての `.mp3` ファイルを `audio/` ディレクトリに配置します。
    - すべての大きな画像ファイルを `image_large/` ディレクトリに配置します。
3.  **プレイリストの設定**:
    - ダウンロードしたJSONファイルのコピーを作成し、ルートディレクトリに `playlist.json` という名前で保存します。
    - `playlist.json` を開き、各トラックの `audio_url` と `image_large_url` のパスを、ローカルファイルを指す相対パスに編集します。
    - 例:
      ```json
      "audio_url": "audio/a5d20ca2-0542-4df9-be56-97f711a81cc7.mp3",
      "image_large_url": "image_large/image_large_a5d20ca2-0542-4df9-be56-97f711a81cc7.jpeg",
      ```
4.  **デプロイ**: プロジェクトフォルダ全体を静的ウェブサーバーでホストするか、GitHub Pagesを使用してデプロイします。

## プレイリストデータ
プレイヤーは `playlist.json` ファイルによって動作します。これはSuno AIからエクスポートしたプレイリストを調整したものです。プレイヤーは以下のフィールドを使用します:
- `name`: プレイリスト/アルバムのタイトル。
- `playlist_clips`: 楽曲オブジェクトの配列。
  - `clip.title`: 楽曲のタイトル。
  - `clip.audio_url`: オーディオファイルへの相対パス。
  - `clip.image_large_url`: アルバムアートへの相対パス。
  - `clip.metadata.prompt`: 楽曲のフル歌詞。
  - `clip.metadata.gpt_description_prompt`: 生成に使用されたプロンプト。楽曲タイトルの作成にも使用されます。
